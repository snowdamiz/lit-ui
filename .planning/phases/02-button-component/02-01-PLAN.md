<task type="auto">
  <name>Task 1: Create button component with variants and sizes</name>
  <files>src/components/button/button.ts</files>
  <action>
Create `src/components/button/button.ts` that:

1. Extends TailwindElement (import from '../../base/tailwind-element')
2. Uses @customElement('ui-button') decorator
3. Defines reactive properties:
   - `variant`: 'primary' | 'secondary' | 'outline' | 'ghost' | 'destructive' (default: 'primary')
   - `size`: 'sm' | 'md' | 'lg' (default: 'md')
   - `disabled`: boolean (default: false)

4. Renders a native `<button>` element with dynamic Tailwind classes

5. Implements variant classes using design tokens from tailwind.css:
   - primary: bg-primary text-primary-foreground hover:opacity-90
   - secondary: bg-secondary text-secondary-foreground hover:bg-accent
   - outline: border border-border bg-transparent text-foreground hover:bg-accent
   - ghost: bg-transparent text-foreground hover:bg-accent
   - destructive: bg-destructive text-destructive-foreground hover:opacity-90

6. Implements size classes:
   - sm: px-3 py-1.5 text-sm
   - md: px-4 py-2 text-base
   - lg: px-6 py-3 text-lg

7. Common base classes for all buttons:
   - inline-flex items-center justify-center
   - rounded-md font-medium
   - transition-colors duration-150
   - cursor-pointer (when not disabled)

8. Disabled state styling:
   - opacity-50 cursor-not-allowed
   - Apply aria-disabled attribute (not HTML disabled - per RESEARCH.md accessibility guidance)

9. Inner glow focus ring (per CONTEXT.md decision):
   - Use static styles with css`` to add button:focus-visible rule
   - outline: none; box-shadow: inset 0 0 0 2px var(--color-ring);

10. Add TypeScript global interface declaration for HTMLElementTagNameMap

Note: Do NOT add form participation, loading state, or icon slots yet - those are in later plans.
