**Props data arrays:**

`radioProps: PropDef[]` — 5 props:
- value (string, "") — The value submitted when this radio is selected
- checked (boolean, false) — Whether this radio is currently selected
- disabled (boolean, false) — Whether this radio is disabled
- label (string, "") — Label text displayed next to the radio
- size ("sm" | "md" | "lg", "md") — Size variant

`radioGroupProps: PropDef[]` — 6 props:
- name (string, "") — Form submission name for the group
- value (string, "") — Currently selected radio value
- required (boolean, false) — Whether a selection is required
- disabled (boolean, false) — Disables all child radios
- label (string, "") — Group label text (rendered as fieldset legend)
- error (string, "") — Error message displayed below the group

   - Arrow Up/Down/Left/Right: Move focus AND selection through options (wrapping)

   - Space: Select focused radio (if not already selected)

**Events to document:**
- `ui-radio-change` on lui-radio: Internal event consumed by RadioGroup (not for consumer use)
- `ui-change` on lui-radio-group: `{ value: string }` — Fired when selection changes

**IMPORTANT:** Radio is ALWAYS used inside RadioGroup. Individual `<lui-radio>` elements are not useful standalone. Structure examples to always show RadioGroup wrapping Radio elements. This is different from Checkbox where individual checkboxes work standalone.
