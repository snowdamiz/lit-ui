**Props tables (from accordion.ts and accordion-item.ts source):**

Accordion props:
| name | type | default | description |
|------|------|---------|-------------|
| value | string | '""' | Comma-separated list of expanded item values (controlled mode) |
| default-value | string | '""' | Initial expanded items for uncontrolled mode |
| multiple | boolean | 'false' | Allow multiple panels open simultaneously |
| collapsible | boolean | 'false' | Allow all panels to close in single-expand mode |
| disabled | boolean | 'false' | Disable all child items |

AccordionItem props:
| name | type | default | description |
|------|------|---------|-------------|
| value | string | '""' | Unique identifier for this item within the accordion group |
| expanded | boolean | 'false' | Whether this item is expanded (set by parent) |
| disabled | boolean | 'false' | Whether this item is disabled |
| heading-level | number | '3' | ARIA heading level for the header element |
| lazy | boolean | 'false' | Defer panel content rendering until first expand |

Accordion slots: `(default)` - Child lui-accordion-item elements
AccordionItem slots: `header` - Content for the accordion header button, `(default)` - Panel content

**CSS custom properties table:** Include all 13 accordion variables from the research (--ui-accordion-border through --ui-accordion-ring) with their default values.

**Interactive demos to include (each with ExampleBlock + code examples for html, react, vue, svelte):**

1. **Basic Accordion** - Single-expand with 3 items, one default-expanded
2. **Multi-expand** - `multiple` attribute with 3 items
3. **Collapsible** - `collapsible` attribute allowing all panels to close
4. **Disabled** - Accordion with `disabled` on container or individual items
5. **Custom Heading Level** - `heading-level="4"` on items
6. **Lazy Content** - Items with `lazy` attribute

**Accessibility section:** Cover keyboard nav (Arrow Up/Down, Home/End, Enter/Space), aria-expanded, heading semantics, focus management.

**PrevNextNav:**
```tsx

  prev={{ title: 'Getting Started', href: '/getting-started' }}
  next={{ title: 'Button', href: '/components/button' }}
/>
```

**Page structure follows TooltipPage exactly:** FrameworkProvider wrapper > div container > header section > demo sections > accessibility section > custom styling section with CSS variables table > API reference with PropsTable + SlotsTable > PrevNextNav.

Export as `export function AccordionPage()`.
