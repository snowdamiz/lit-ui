Note: description fields can contain HTML angle brackets like `<lui-button>` — these are
inside the frontmatter block which gets stripped entirely. The regex `/^---[\s\S]*?---\n/`
(non-greedy, dotall) handles multi-line description fields correctly.

Create `scripts/compile-knowledge.ts` using only Node.js built-ins. Implement the following:

**Path constants (from scripts/ directory, one level up to project root):**
```typescript
const SKILL_DIR = path.join(__dirname, '..', 'skill')
const SKILLS_DIR = path.join(SKILL_DIR, 'skills')
const OUTPUT_PATH = path.join(SKILL_DIR, 'lit-ui-knowledge.xml')
```

**Helper functions (implement all four):**

1. `stripFrontmatter(content: string): string`
   - Strip the `---...---\n` block at the start using `/^---[\s\S]*?---\n/`
   - The trailing `\n` is required — omitting it causes frontmatter bleed into first section

2. `xmlEscape(text: string): string`
   - Replace in order: `&` → `&amp;` FIRST, then `<` → `&lt;`, then `>` → `&gt;`, then `"` → `&quot;`
   - CRITICAL: `&` must be replaced first — reversing order produces double-encoded sequences like `&amp;lt;`

3. `extractSkillName(markdown: string): string`
   - Match `/^# (.+)$/m` — H1 heading becomes the `<skill name="...">` attribute value
   - Return `'unknown'` if no match

4. `extractSections(markdown: string): Array<{title: string; content: string}>`
   - Split on `##` headings using `/^(?=## )/m` lookahead — CRITICAL: use lookahead or the `## ` prefix gets consumed and title extraction fails
   - For each part, match `/^## (.+)\n([\s\S]*)/` to extract title and content
   - `.trim()` the content to avoid leading/trailing whitespace
   - Parts before the first `##` (preamble/H1) are skipped — do NOT create a section for the H1

5. `buildSkillXml(name: string, sections: Array<{title: string; content: string}>): string`
   - Build `<skill name="${xmlEscape(name)}">` wrapper
   - Each section: `  <section title="${xmlEscape(s.title)}">\n${xmlEscape(s.content)}\n  </section>`
   - Join sections with `\n`

**Main function `compileKnowledge()`:**
1. Read `skill/SKILL.md` as router skill (always first)
2. `readdir(SKILLS_DIR)` → `.sort()` for deterministic order → read each `skills/{dir}/SKILL.md`
3. For each file: `stripFrontmatter` → `extractSkillName` → `extractSections`
4. Build XML document:
   ```
   <?xml version="1.0" encoding="UTF-8"?>
   <skills>
     <skill name="...">           (each indented 2 spaces)
       <section title="...">
   ...content...
       </section>
     </skill>
   </skills>
                                  (trailing newline)
   ```
   Indent each skill element by prepending `  ` to each line.
5. `fs.writeFile(OUTPUT_PATH, xml, 'utf8')`
6. `console.log(`Compiled ${1 + subDirs.length} skills → ${OUTPUT_PATH}`)`

Call `await compileKnowledge()` at top level (ESM top-level await — no IIFE needed).

Run invocation: `node --experimental-strip-types scripts/compile-knowledge.ts`
