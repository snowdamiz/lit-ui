**FormIntegrationGuide.tsx** - Create following guide page structure.

**Sections:**

1. **Overview** - All date/time components use ElementInternals for native form participation. They work like native inputs inside `<form>` elements.

2. **Value Formats** - Table showing each component's form submission format:
   - Calendar: YYYY-MM-DD (ISO 8601 date)
   - Date Picker: YYYY-MM-DD (ISO 8601 date)
   - Date Range Picker: YYYY-MM-DD/YYYY-MM-DD (ISO 8601 interval)
   - Time Picker: HH:mm:ss (ISO 8601 time)

3. **Basic Form Example** - CodeBlock showing a form with date-picker and time-picker, demonstrating FormData retrieval:
```html
<form id="booking-form">
  <lui-date-picker name="date" label="Date" required></lui-date-picker>
  <lui-time-picker name="time" label="Time" required></lui-time-picker>
  <lui-button type="submit">Book</lui-button>
</form>
<script>
  document.getElementById('booking-form').addEventListener('submit', (e) => {
    e.preventDefault();
    const data = new FormData(e.target);
    console.log(data.get('date')); // "2026-02-14"
    console.log(data.get('time')); // "14:30:00"
  });
</script>
```

4. **Validation** - Document validation states:
   - `required` attribute triggers valueMissing
   - Min/max constraints trigger rangeUnderflow/rangeOverflow
   - Custom error messages via `error` attribute
   - Form reset via `formResetCallback()` clears values

5. **React/Vue/Svelte Integration** - Framework-specific code examples showing form handling in each framework.

**I18nGuide.tsx** - Create following guide page structure.

**Sections:**

1. **Overview** - All date/time components support internationalization via the `locale` attribute using BCP 47 locale tags and the native Intl API.

2. **Setting Locale** - Show how to set locale on each component:
```html
<lui-calendar locale="de-DE"></lui-calendar>
<lui-date-picker locale="ja-JP" label="Date"></lui-date-picker>
<lui-date-range-picker locale="fr-FR" label="Period"></lui-date-range-picker>
<lui-time-picker locale="en-GB" label="Time"></lui-time-picker>
```

3. **What Locale Affects** - Table of locale-dependent behaviors:
   - Month and weekday names (Intl.DateTimeFormat)
   - First day of week (Intl.Locale.getWeekInfo())
   - Date display format (MM/dd vs dd/MM for date picker)
   - 12h/24h default (auto-detected from locale for time picker)
   - Number formatting

4. **Overriding Locale Defaults** - Show how to override specific behaviors:
   - `first-day-of-week` attribute overrides locale's first day
   - `hour12` attribute overrides locale's hour cycle
   - `format` JS property overrides display format

5. **Supported Locales** - Note that any BCP 47 locale supported by the browser's Intl API is supported. No bundled locale data.

6. **RTL Support** - Note any RTL considerations for Arabic, Hebrew locales.
