  <action>
Add dark mode overrides for the utility classes defined in index.css.

Add these `.dark` selector overrides AFTER the existing utility class definitions:

```css
/* Dark mode overrides for utility classes */
.dark .grid-pattern {
  background-image:
    linear-gradient(to right, oklch(0.18 0 0) 1px, transparent 1px),
    linear-gradient(to bottom, oklch(0.18 0 0) 1px, transparent 1px);
}

.dark .hero-gradient {
  background: radial-gradient(ellipse 80% 50% at 50% -20%, oklch(0.15 0 0) 0%, transparent 70%);
}

.dark .bottom-fade {
  background: linear-gradient(to bottom, transparent 0%, oklch(0.10 0 0) 100%);
}

.dark::selection {
  background-color: oklch(0.85 0 0);
  color: oklch(0.10 0 0);
}

.dark .focus-ring:focus-visible {
  box-shadow: 0 0 0 2px oklch(0.10 0 0), 0 0 0 4px oklch(0.85 0 0);
}

.dark .text-gradient {
  background: linear-gradient(135deg, oklch(0.95 0 0) 0%, oklch(0.65 0 0) 100%);
}

.dark .divider-fade {
  background: linear-gradient(to right, transparent, oklch(0.25 0 0), transparent);
}

.dark .card-elevated {
  box-shadow:
    0 1px 3px 0 oklch(0 0 0 / 0.2),
    0 1px 2px -1px oklch(0 0 0 / 0.2);
}

.dark .card-elevated:hover {
  box-shadow:
    0 10px 15px -3px oklch(0 0 0 / 0.3),
    0 4px 6px -4px oklch(0 0 0 / 0.3);
}
```

Note: The code-block class already has dark styling (dark background) - no changes needed.
Note: Terminal glow effect works on both modes since it's subtle.
